APIs & Endpoints

Customer-Facing APIs

GET
/api/campaigns/active

Purpose: Get list of active campaigns for display

Response: Array of active campaign objects with details

POST
/api/vouchers/generate

Purpose: Create new voucher with customer data

Request: campaign_id, customer_data, facebook_verified

Response: voucher_code, expiry_date, success_status

Admin Management APIs

POST
/api/admin/campaigns

Purpose: Create new promotional campaign

Request: Campaign details, timing, discount info

Response: campaign_id, success_status

PUT
/api/admin/vouchers/{code}/redeem

Purpose: Mark voucher as redeemed

Request: staff_id, redemption_notes

Response: success_status, updated_voucher

Analytics & Export APIs

GET
/api/analytics/campaigns/{id}

Purpose: Get campaign performance metrics

Response: Total vouchers, redemption rate, revenue impact

POST
/api/export/sheets

Purpose: Export voucher data to Google Sheets

Request: campaign_id, date_range

Response: sheet_url, export_status

üîê Authentication & Authorization

  • Role-Based Access: Separate permissions for customers vs admin staff
  • Firebase Security Rules: Database-level protection for all operations
  • Anonymous Auth: Customers can generate vouchers without account creation
  • Staff Verification: Email-based authentication for admin access

üîë Data Protection

  • Secure Voucher Codes: Unique, non-guessable codes with expiration
  • Customer Data Privacy: Optional data collection with clear consent
  • IP Tracking: Basic fraud detection for voucher generation
  • Encrypted Storage: Firebase automatic encryption for all data

üõ°Ô∏è Privacy Measures
